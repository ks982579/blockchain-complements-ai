## Overview

This paper presents a UAV surveillance platform that integrates blockchain, Non-Fungible Tokens (NFTs), Federated Learning (FL), and Zero-Trust (ZT) security to address the challenges of operating UAV fleets within 5G/6G network-sliced environments. Traditional UAV monitoring relies on direct, unlicensed-spectrum communication with ground pilots, which is low-bandwidth, insecure, and range-limited; moving UAVs onto cellular network slices improves performance but introduces new problems around UAV identity management, access control, and communication permission management across slices. The paper's core contribution is an architecture in which each UAV's attributes are represented as an NFT (using a custom "i-528" schema), UAV identities and communication permissions are enforced via blockchain smart contracts under a Zero-Trust model, and a coordinator-less Federated Learning system runs on top of a sharded blockchain (one shard per network slice) to build ML models from UAV sensor data without centralizing raw data or relying on a single trusted aggregator. As a concrete use case, the system trains a GRU-based federated model to detect anomalous UAV network communications and identify attack surfaces, with model provenance recorded as Model Card objects on the blockchain ledger. The platform was implemented and evaluated on a real 5G testbed using a FreedomFi 5G gateway, Indoor Radio Cell, Raspberry Pi-simulated UAV sensors, and the Moose blockchain with Aplos smart contracts, demonstrating feasibility through measurements of federated model accuracy/loss and blockchain block-generation time.

## Background

The paper situates its work within several established lines of research and technology that it draws on: 5G/6G network slicing as a mechanism for running multiple logical networks over shared infrastructure to meet differing UAV communication requirements (citing prior NFV-SDN work on edge services); NFTs as a general token standard for representing unique digital assets (citing a survey on NFT overview, evaluation, and challenges); Federated Learning as a privacy-preserving distributed training paradigm (citing Konečný et al.'s foundational FL communication-efficiency work); and Zero-Trust security architectures for 5G/6G networks that assume no implicit trust in any network component (citing work on trust in 5G and beyond). The authors also build heavily on their own prior research stack, including Moose (a sharded, slicing-aware blockchain architecture for 5G IoT), Aplos (a smart contract framework), Rahasak (an enterprise blockchain architecture), Skunk (a blockchain/Zero-Trust FL platform for 5G/6G slicing), Casper (a blockchain-based credential verification system), Octopus (a privacy-preserving P2P transaction system using IPFS), Kaputa (an NFT/model-card-integrated 5G/6G network slice marketplace), Bassa-ML (a blockchain and model-card FL provenance platform), and Indy528 (the origin of the i-528 NFT schema for tokenizing FL models). It further relies on Model Card concepts for ML model transparency and provenance (citing Wadhwani & Jain's model card toolkit work and the authors' own "Let's-Trace" supply-chain provenance platform), Apache Kafka for local consensus, and Raft for cross-shard/global consensus. The related-work section also surveys comparable blockchain-based federated learning systems—DeepChain, BlockFL, ModelChain, Poster, BAFFLE, Chained-FL, Breaking Bad, and TMLC—to position the proposed platform's feature set (Zero-Trust support, FL, auditing/provenance, model cards, real-time predictions) against existing alternatives.

## Key Points

- The paper proposes a combined blockchain + NFT + Federated Learning + Zero-Trust security platform specifically designed for UAV monitoring in 5G/6G network-sliced environments, addressing UAV identity, access control, and communication permission management challenges that arise when UAVs are integrated into cellular slices.
- It introduces a sharding-based blockchain consensus architecture where each 5G network slice hosts its own blockchain shard with independent local consensus (via Apache Kafka) and optional global cross-shard consensus (via Raft), improving scalability and reducing transaction processing time.
- UAV attributes/identities are encoded as NFT tokens using a custom schema called "i-528," enabling structured, standardized, and interoperable representation of each UAV's unique characteristics within the platform.
- The platform's local shard architecture is organized into five layers: a Blockchain storage layer (storing UAV identities, ZT policy rules, FL models, and provenance data), a Smart Contract layer (identity, sensor, model, and permission contracts), a Federated Learning layer (the FML service), an Off-chain storage layer (per-peer sensitive sensor data, hashed and referenced on-chain), and a 5G core network layer (NSSF-based network slicing, NFV/SDN microservices).
- The proposed Federated Learning system is coordinator-less, eliminating the single-point-of-failure/attack risk of a centralized FL aggregator while still allowing cross-shard aggregation of local models (via Stochastic Gradient Descent-based model averaging) into a global model.
- Generated ML models are stored as Model Card objects directly on the blockchain ledger, providing auditable provenance information (participating peers, model parameters, training information) that any party can verify.
- A Zero-Trust policy enforcement mechanism uses a rule engine to continuously verify UAV sensor communications (such as allowed ports and endpoints) against rules encoded and stored on the blockchain ledger.
- The paper demonstrates a concrete use case—anomaly/attack-surface detection in UAV network traffic using a Gated Recurrent Unit (GRU) federated model trained across blockchain peers, each with local IoT datasets—integrated into smart contracts for real-time validation of incoming sensor data.
- The system was implemented on a real 5G testbed (FreedomFi 5G gateway, Indoor Radio Cell, Raspberry Pi UAV sensor simulators, AWS-hosted blockchain services, Moose blockchain ledger, Aplos smart contracts) and evaluated for federated model accuracy/training loss over 1000 iterations and blockchain block-generation time (e.g., 8.5 seconds for a 10,000-transaction block), with additional measurements across varying peer counts and block sizes.
- The paper provides a comparative feature table positioning its platform against existing blockchain-based decentralized learning systems (DeepChain, Breaking Bad, TMLC, BlockFL, ModelChain, BAFFLE, Poster, Chained-FL), claiming it is the only one offering Zero-Trust support, Federated Learning, auditing/provenance, model card reporting, and real-time predictions simultaneously within a 5G application domain.

## Conclusion

The paper concludes that combining blockchain, NFTs, Zero-Trust security, and coordinator-less Federated Learning into a sharded, slice-aware architecture is a feasible approach for secure, scalable, and decentralized UAV monitoring in 5G/6G networks. The reported evaluation (federated model accuracy/loss trends across peers and iterations, and block generation time measurements under varying peer counts and block/transaction sizes) is presented as evidence supporting the system's practical viability, and the use case shows the platform can detect anomalous UAV communications and attack surfaces via FL-trained GRU models with on-chain model card provenance. However, the evaluation is largely a feasibility/performance demonstration on a small testbed (Raspberry Pi-simulated UAVs) rather than a rigorous comparison against baseline systems or an adversarial security evaluation of the Zero-Trust and NFT components themselves; the paper does not report quantitative security analysis (e.g., resistance to specific attacks) or scalability limits beyond the tested peer/block-size ranges. The authors explicitly flag future work as integrating TensorFlow-Federated libraries into the platform, suggesting the current FL implementation is custom/lightweight rather than built on a mature FL framework — an area for further investigation, along with broader empirical validation of the sharding consensus approach at larger scale and deeper assessment of the NFT-based identity/access-control model's robustness against real-world adversarial UAV network conditions.
